Pattern Overviewنظرة عامة على الباترن

The Decorator pattern wraps an object to add behavior without changing its class. Both the wrapper and the wrapped object implement the same interface, so they're interchangeable.باترن Decorator بيلف object عشان يضيف سلوك من غير ما يغير كلاسه. الـ wrapper والـ object المغلّف بينفذوا نفس الـ interface، عشان كده بيكونوا قابلين للتبادل.

Key Idea: Logger wraps Cache wraps Database. Each layer adds a responsibility, all exposing the same interface.الفكرة الأساسية: Logger بيلف Cache بيلف Database. كل طبقة بتضيف مسؤولية، وكلهم بيعرضوا نفس الـ interface.

Real Problem Exampleمثال على مشكلة حقيقية

You have a UserRepository querying the database. You want to add caching AND logging. Instead of modifying it, create CachingUserRepository and LoggingUserRepository decorators that wrap it.عندك UserRepository بتستعلم من الـ database. عايز تضيف caching وlogging. بدل ما تعدل عليه، انشئ CachingUserRepository وLoggingUserRepository كـ decorators بتلفه.

Laravel Implementationالتطبيق في Laravel

Laravel's Middleware is the most direct application of Decorator. Each middleware wraps the request handler, adding behavior (auth, CORS, throttling) before and after.الـ Middleware في Laravel هو أوضح تطبيق لـ Decorator. كل middleware بيلف الـ request handler ويضيف سلوك (auth, CORS, throttling) قبل وبعد.

Laravel's Pipeline class also implements Decorator chaining — used by the Router and HTTP Kernel for middleware stacks.كلاس Pipeline في Laravel كمان بينفذ Decorator chaining — بيستخدمه الـ Router وHTTP Kernel لـ middleware stacks.

Laravel Code Exampleمثال كود Laravel

app/Repositories/CachingUserRepository.php
<?php
namespace App\Repositories;
use App\Contracts\UserRepositoryInterface;
use Illuminate\Support\Facades\Cache;

// Decorator: adds caching around any UserRepository
// Decorator: بيضيف caching حول أي UserRepository
class CachingUserRepository implements UserRepositoryInterface
{
    public function __construct(
        private UserRepositoryInterface $inner, // wrapped repo / الـ repo المغلّف
        private int $ttl = 3600
    ) {}

    public function find(int $id): ?User
    {
        return Cache::remember("user:{$id}", $this->ttl, function () use ($id) {
            return $this->inner->find($id); // delegate / فوّض للـ inner
        });
    }
    public function findByEmail(string $email): ?User
    {
        return Cache::remember('user:email:' . md5($email), $this->ttl,
            fn() => $this->inner->findByEmail($email));
    }
    public function all(): \Illuminate\Support\Collection
    {
        return Cache::remember('users:all', $this->ttl, fn() => $this->inner->all());
    }
}
Stacking Decorators / تكديس الـ Decorators
<?php
// Stack decorators: Log → Cache → Database
// كدّس الـ decorators: Log → Cache → Database
$this->app->bind(UserRepositoryInterface::class, function () {
    $base   = new EloquentUserRepository();        // core
    $cached = new CachingUserRepository($base);    // + caching
    $logged = new LoggingUserRepository($cached);  // + logging
    return $logged;
    // Call chain: logged->find() → cached->find() → eloquent->find()
});

// Laravel Middleware = Decorator chain
// Middleware في Laravel = سلسلة Decorator
// Auth → Throttle → ValidateCsrf → YourController
// كل middleware بيلف اللي بعده ويضيف سلوك

When to Use Itمتى تستخدمه

✓ Good Use Cases✓ حالات جيدة

  • Adding caching to a repository
  • Adding logging without modifying core logic
  • Middleware pipelines, retry logic, auditing
  • إضافة caching لـ repository
  • إضافة logging من غير تعديل الـ core logic
  • Middleware pipelines وretry logic والـ auditing

✗ Avoid When✗ تجنبه عندما

  • Simple subclassing would be cleaner
  • Too many decorator layers make debugging hard
  • الـ subclassing البسيط سيكون أنظف
  • طبقات decorator كتير بتصعّب الـ debugging

When NOT to Use Itمتى لا تستخدمه

Decorators can make stack traces confusing with many layers. For cross-cutting concerns in Laravel, consider Middleware (HTTP), Events/Listeners (side effects), or Eloquent Observers (model events) first.الـ Decorators ممكن تخلي الـ stack traces محيّرة مع كتر الطبقات. للـ cross-cutting concerns في Laravel، فكر في Middleware (HTTP) أو Events/Listeners (side effects) أو Eloquent Observers (model events) الأول.
