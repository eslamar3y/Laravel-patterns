Pattern Overviewنظرة عامة على الباترن

The Command pattern encapsulates a complete request (action + data) as an object. This object can be stored, queued, serialized, logged, retried, or undone.باترن Command بيغلّف request كامل (action + بيانات) كـ object. الـ object ده ممكن يتخزن، يتحط في queue، يتسلسل، يتسجّل، يُعاد تنفيذه، أو يُلغى.

Key Parts: Command = object with data + execute logic. Invoker = the dispatcher. Receiver = the handler that does the work.الأجزاء الرئيسية: Command = object بالبيانات + منطق التنفيذ. Invoker = الـ dispatcher. Receiver = الـ handler اللي بيعمل الشغل.

Real Problem Exampleمثال على مشكلة حقيقية

Sending an invoice after checkout is slow and can fail. You need it to run in background, be retryable, and delayable. Wrapping it as a Command (Job) solves all of these.إرسال invoice بعد الـ checkout بطيء وممكن يفشل. محتاجه يتنفذ في الخلفية، قابل للإعادة، وقابل للتأجيل. تغليفه كـ Command (Job) بيحل كل ده.

Laravel Implementationالتطبيق في Laravel

Laravel Jobs are a direct, production-ready Command Pattern implementation. Each Job is a command class with constructor data and a handle() method. The Queue is the Invoker.Jobs في Laravel هي تطبيق مباشر وجاهز للإنتاج لـ Command Pattern. كل Job هي command class بـ constructor data وmethod handle(). الـ Queue هو الـ Invoker.

Artisan Commands are also Command pattern — each console command encapsulates an operation with its input. The Bus routes commands.Artisan Commands هي كمان Command pattern — كل console command بتغلّف عملية مع المدخلات بتاعتها. الـ Bus بيوجّه الـ commands.

Laravel Code Exampleمثال كود Laravel

app/Jobs/SendInvoiceEmail.php
<?php
namespace App\Jobs;
use App\Models\{User, Order};
use App\Mail\InvoiceMail;
use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Foundation\Bus\Dispatchable;
use Illuminate\Queue\{InteractsWithQueue, SerializesModels};

// Command: encapsulates the complete "send invoice" operation
// Command: بتغلف عملية "إرسال invoice" الكاملة
class SendInvoiceEmail implements ShouldQueue
{
    use Dispatchable, InteractsWithQueue, Queueable, SerializesModels;

    public int $tries   = 3;  // retry on failure / إعادة المحاولة عند الفشل
    public int $backoff = 60; // wait 60s between tries / انتظر 60 ثانية بين المحاولات

    public function __construct(
        public readonly Order $order,
        public readonly User  $user,
    ) {}

    // Receiver: does the actual work / المستقبل: بيعمل الشغل الفعلي
    public function handle(): void
    {
        \Mail::to($this->user)->send(new InvoiceMail($this->order));
        $this->order->update(['invoice_sent_at' => now()]);
    }

    // Called after all retries exhausted / بيتستدعي بعد انتهاء كل المحاولات
    public function failed(\Throwable $exception): void
    {
        \Log::error("Invoice email failed: {$exception->getMessage()}");
    }
}
Dispatching Commands / إطلاق الـ Commands
<?php
// Dispatch to queue (async) / إرسال للـ queue (async)
SendInvoiceEmail::dispatch($order, $user);

// With delay / مع تأجيل
SendInvoiceEmail::dispatch($order, $user)->delay(now()->addMinutes(5));

// On specific queue/connection / على queue/connection محدد
SendInvoiceEmail::dispatch($order, $user)
    ->onQueue('emails')->onConnection('redis');

// Synchronously (for testing) / بشكل متزامن (للاختبار)
SendInvoiceEmail::dispatchSync($order, $user);

// Chain commands — run in sequence / سلسلة commands — تتنفذ بالتسلسل
\Bus::chain([
    new ProcessPayment($order),
    new SendInvoiceEmail($order, $user),
    new UpdateInventory($order),
])->dispatch();

When to Use Itمتى تستخدمه

✓ Good Use Cases✓ حالات جيدة

  • Background processing (emails, reports, exports)
  • Operations that need retry logic
  • CQRS architecture
  • Scheduled or delayed operations
  • معالجة خلفية (emails, reports, exports)
  • عمليات محتاجة retry logic
  • معمارية CQRS
  • عمليات مجدولة أو مؤجلة

✗ Avoid When✗ تجنبه عندما

  • Simple synchronous operations
  • Overhead of queuing not justified
  • You need an immediate return value
  • عمليات synchronous بسيطة
  • حمل الـ queuing مش مبرر
  • محتاج قيمة إرجاع فورية

When NOT to Use Itمتى لا تستخدمه

Not every operation needs a Job. Simple CRUD doesn't need a command bus. The pattern shines for background processing, retry, scheduling, or undo. For simple operations, direct service calls are cleaner.مش كل عملية محتاجة Job. الـ CRUD البسيط مش محتاج command bus. الباترن بيتألق في الـ background processing والـ retry والـ scheduling والـ undo. للعمليات البسيطة، استدعاء الـ service مباشرة أنظف.
