SingletonSingleton

Ensure a class has only one instance and provide a global access point to it.ضمان وجود instance واحدة فقط من الكلاس وتوفير نقطة وصول global إليها.

Pattern Overviewنظرة عامة على الباترن

The Singleton pattern restricts the instantiation of a class to a single object. No matter how many times you ask for an instance, you always get back the same one.باترن Singleton بيقيّد إنشاء الـ class لـ object واحد بس. مهما طلبت الـ instance، دايمًا بترجعلك نفس الواحدة.

It solves the problem of having multiple conflicting instances of something that should be unique — like a database connection, a configuration store, or a logger.بيحل مشكلة وجود instances متعددة متضاربة لحاجة المفروض تكون unique — زي database connection أو config store أو logger.

Key Idea: One class → one instance → global access point.الفكرة الأساسية: كلاس واحد ← instance واحدة ← نقطة وصول global.

Real Problem Exampleمثال على مشكلة حقيقية

Imagine your app needs to read configuration values. Without Singleton, every class that needs config would create its own Config object, reading files from disk each time — wasteful and inconsistent.تخيّل إن تطبيقك محتاج يقرأ الـ configuration. من غير Singleton، كل كلاس محتاج config هينشئ Config object جديد ويقرأ من الـ disk كل مرة — مضيعة وغير متسق.

With Singleton, the Config object is created once and every part of the app shares the same instance. No duplicate reads, no inconsistency.مع Singleton، الـ Config object بيتنشأ مرة واحدة وكل التطبيق بيشارك نفس الـ instance. مفيش قراءة مكررة ومفيش تناقض.

Laravel Implementationالتطبيق في Laravel

Laravel\'s Service Container implements Singleton via the singleton() method in Service Providers. Once bound, the container creates the object once and returns the same instance every time.الـ Service Container في Laravel بينفذ Singleton عبر الـ method singleton() في الـ Service Providers. بعد الـ binding، الـ container بينشئ الـ object مرة واحدة ويرجع نفس الـ instance في كل مرة.

Core Laravel services like Config, Auth, DB, Cache, and Log are all registered as singletons.الـ services الأساسية في Laravel زي Config وAuth وDB وCache وLog كلها متسجلة كـ singletons.

Laravel Tip: Use $this->app->singleton() for one instance, $this->app->bind() for a new instance every time.نصيحة: استخدم singleton() لـ instance واحدة، وbind() لـ instance جديدة في كل مرة.

Laravel Code Exampleمثال كود Laravel

app/Services/ConfigManager.php
<?php

namespace App\Services;

class ConfigManager
{
    private array $config = [];

    public function __construct()
    {
        // Expensive operation — only runs ONCE / بتتنفذ مرة واحدة بس
        $this->config = [
            'app_name'   => config('app.name'),
            'app_env'    => config('app.env'),
            'debug_mode' => config('app.debug'),
        ];
    }

    public function get(string $key, mixed $default = null): mixed
    {
        return $this->config[$key] ?? $default;
    }
}
app/Providers/AppServiceProvider.php 
<?php

namespace App\Providers;

use App\Services\ConfigManager;
use Illuminate\Support\ServiceProvider;

class AppServiceProvider extends ServiceProvider
{
    public function register(): void
    {
        // Singleton = one instance for the whole request
        // Singleton = instance واحدة طول الـ request
        $this->app->singleton(ConfigManager::class, function ($app) {
            return new ConfigManager();
        });
    }
}
app/Http/Controllers/HomeController.php 
<?php

namespace App\Http\Controllers;

use App\Services\ConfigManager;

class HomeController extends Controller
{
    public function __construct(
        private ConfigManager $configManager
    ) {}

    public function index()
    {
        // Same instance every time / نفس الـ instance في كل مكان
        $appName = $this->configManager->get('app_name');
        return view('home', compact('appName'));
    }
}
singleton() vs bind(): bind() creates a new instance every call. singleton() creates once and caches. instance() binds an already-created object.singleton() مقابل bind(): bind() بتنشئ instance جديدة في كل call. singleton() بتنشئ مرة واحدة وتحفظها. instance() بتربط object موجود مسبقاً.

When to Use Itمتى تستخدمه

✓ Good Use Cases✓ حالات جيدة

  • Database connection pools
  • Application configuration stores
  • Logging services
  • Cache managers
  • API client services
    •
  • Connection pools للـ database
  • مخازن الـ configuration
  • خدمات الـ logging
  • مديري الـ cache
  • API clients

✗ Avoid When✗ تجنبه عندما

  • Object has user-specific state
  • You need multiple instances per request
  • Object holds mutable shared state
    •
  • الـ object عنده state خاصة بالـ user
  • محتاج instances متعددة في الـ request
  • الـ object بيحتفظ بـ mutable shared state

When NOT to Use Itمتى لا تستخدمه

The Singleton is often called an anti-pattern when overused. Don\'t reach for it just because something "feels global."الـ Singleton بيُعتبر anti-pattern لما يُستخدم بإفراط. ما تستخدمهوش بس لأن حاجة "حاسس إنها global."

Warning: In Laravel Octane (Swoole/RoadRunner), singletons persist between requests. Never store request-specific data in them.تحذير: في Laravel Octane، الـ singletons بتفضل موجودة بين الـ requests. ما تحطش فيها بيانات خاصة بـ request معين.
