AdapterAdapter

Convert one interface into another that clients expect to use.تحويل interface لـ interface تاني يتوقعه الـ client.

Pattern Overviewنظرة عامة على الباترن

The Adapter pattern converts the interface of one class into an interface that another class expects. Like a power adapter that lets your laptop work in a foreign outlet.باترن Adapter بيحوّل الـ interface بتاع كلاس لـ interface تاني بيتوقعه كلاس آخر. زي adapter الكهرباء اللي بيخلي شاحن اللاب توب يشتغل في مقبس أجنبي.

Key Idea: You have class A with sendMessage() and need it to work where dispatch() is expected. The adapter wraps A and implements the expected interface.الفكرة الأساسية: عندك كلاس A عنده sendMessage() ومحتاجه يشتغل في مكان بيتوقع dispatch(). الـ Adapter بيلف A وبينفذ الـ interface المطلوب.

Real Problem Exampleمثال على مشكلة حقيقية

Your app uses an SMS service with sendSms($to, $body). You want to switch to Twilio which has a completely different SDK. Instead of rewriting everything, you write an adapter.تطبيقك بيستخدم SMS service عندها sendSms($to, $body). عايز تتحول لـ Twilio اللي عنده SDK مختلف خالص. بدل ما تعيد كتابة كل حاجة، بتكتب adapter.

Laravel Implementationالتطبيق في Laravel

Laravel's Cache and Filesystem systems use Adapter. Each driver (Redis, Memcached, File, S3) wraps a third-party client and implements Laravel's unified interface.أنظمة Cache وFilesystem في Laravel بتستخدم Adapter. كل driver (Redis, Memcached, File, S3) بيلف third-party client وبينفذ الـ unified interface بتاع Laravel.

Flysystem: Laravel wraps the Flysystem library so you always call Storage::put() whether writing to local disk or S3.Flysystem: Laravel بيلف مكتبة Flysystem عشان دايمًا تستدعي Storage::put() سواء بتكتب على الـ disk المحلي أو S3.

Laravel Code Exampleمثال كود Laravel

app/Contracts/SmsProviderInterface.php
<?php
namespace App\Contracts;
// Our app's expected interface / الـ interface اللي تطبيقنا بيتوقعه
interface SmsProviderInterface
{
    public function sendSms(string $to, string $body): bool;
}
app/Adapters/TwilioSmsAdapter.php
<?php
namespace App\Adapters;
use App\Contracts\SmsProviderInterface;
use Twilio\Rest\Client as TwilioClient;

// Adapter wraps Twilio and exposes our interface
// الـ Adapter بيلف Twilio ويعرض interface بتاعنا
class TwilioSmsAdapter implements SmsProviderInterface
{
    public function __construct(private TwilioClient $twilio) {}

    public function sendSms(string $to, string $body): bool
    {
        try {
            $this->twilio->messages->create($to, [
                'from' => config('services.twilio.from'),
                'body' => $body,
            ]);
            return true;
        } catch (\Exception $e) {
            \Log::error('SMS failed: ' . $e->getMessage());
            return false;
        }
    }
}
Binding & Usage / الربط والاستخدام
<?php
// Bind adapter in ServiceProvider
$this->app->singleton(SmsProviderInterface::class, function () {
    $twilio = new \Twilio\Rest\Client(
        config('services.twilio.sid'),
        config('services.twilio.token')
    );
    return new TwilioSmsAdapter($twilio);
});

// Service only knows the interface — not Twilio
// الـ Service بيعرف الـ interface بس — مش Twilio
class AuthService {
    public function __construct(private SmsProviderInterface $sms) {}
    public function sendCode(User $user): void {
        $code = rand(100000, 999999);
        $this->sms->sendSms($user->phone, "Your code: {$code}");
    }
}

When to Use Itمتى تستخدمه

✓ Good Use Cases✓ حالات جيدة

  • Integrating third-party SDKs
  • Wrapping legacy code in a modern interface
  • Supporting multiple providers behind one interface
  • دمج SDKs من طرف تالت
  • تغليف الكود القديم بـ interface حديث
  • دعم providers متعددة خلف interface واحد

✗ Avoid When✗ تجنبه عندما

  • You can modify the original class directly
  • The interfaces are already compatible
  • ممكن تعدل الكلاس الأصلي مباشرة
  • الـ interfaces متوافقة أصلاً

When NOT to Use Itمتى لا تستخدمه

Don't adapt everything by default. If you only use one provider and never intend to swap, injecting the client directly is simpler. Introduce Adapter when you need to hide an implementation.ما تعملش adapter لكل حاجة بالـ default. لو بتستخدم provider واحد ومش هتبدله، حقن الـ client مباشرة أبسط. ضيف Adapter لما تحتاج تخبّي الـ implementation.
