StrategyStrategy

Define a family of algorithms and make them interchangeable at runtime.تعريف مجموعة خوارزميات وجعلها قابلة للتبادل وقت التشغيل.

Pattern Overviewنظرة عامة على الباترن

The Strategy pattern defines a family of algorithms, puts each in its own class, and makes them interchangeable. The context uses a strategy without knowing its details.باترن Strategy بيعرّف مجموعة خوارزميات، بيحط كل واحدة في كلاسها، وبيخليهم قابلين للتبادل. الـ context بيستخدم strategy من غير ما يعرف تفاصيلها.

Key Idea: The context says "calculate shipping" — the strategy decides whether to use FedEx rates, UPS rates, or DHL rates.الفكرة الأساسية: الـ context بيقول "احسب الشحن" — الـ strategy هي اللي تقرر هتستخدم أسعار FedEx أو UPS أو DHL.

Real Problem Exampleمثال على مشكلة حقيقية

Your e-commerce app supports FedEx, UPS, and DHL — each with different rate calculation logic. A massive if/else in your OrderService would be hard to maintain. Strategy encapsulates each in its own class.تطبيق التجارة الإلكترونية بيدعم FedEx وUPS وDHL — كل واحد بحسابات أسعار مختلفة. if/else ضخم في OrderService صعب تحافظ عليه. Strategy بتغلف كل واحد في كلاسه.

Laravel Implementationالتطبيق في Laravel

Laravel's driver-based systems are Strategy in action: Cache drivers (redis, file, database), Mail drivers (smtp, ses, mailgun), Queue drivers (redis, sqs) all implement a shared interface.أنظمة الـ drivers في Laravel هي Strategy في العمل: Cache drivers (redis, file, database)، Mail drivers (smtp, ses, mailgun)، Queue drivers (redis, sqs) كلهم بينفذوا shared interface.

You swap strategies via .env config without changing application code.بتبدل الـ strategies عبر config الـ .env من غير تغيير كود التطبيق.

Laravel Code Exampleمثال كود Laravel

app/Contracts/ShippingStrategyInterface.php
<?php
namespace App\Contracts;
interface ShippingStrategyInterface
{
    public function calculateCost(float $weight, string $destination): float;
    public function getEstimatedDays(string $destination): int;
    public function getCarrierName(): string;
}
app/Shipping/FedExStrategy.php + UPSStrategy.php
<?php
class FedExStrategy implements ShippingStrategyInterface
{
    public function calculateCost(float $weight, string $destination): float
    {
        $baseRate = 12.50;
        return $baseRate + ($weight * 0.85) + $this->getZoneSurcharge($destination);
    }
    public function getEstimatedDays(string $destination): int
    {
        return str_starts_with($destination, 'US') ? 2 : 7;
    }
    public function getCarrierName(): string { return 'FedEx'; }
    private function getZoneSurcharge(string $d): float
    {
        return in_array($d, ['AK', 'HI']) ? 15.00 : 0.00;
    }
}

class UPSStrategy implements ShippingStrategyInterface
{
    public function calculateCost(float $weight, string $destination): float
    {
        return 10.00 + ($weight * 1.10); // UPS different rates / أسعار UPS مختلفة
    }
    public function getEstimatedDays(string $destination): int { return 3; }
    public function getCarrierName(): string { return 'UPS'; }
}
ShippingCalculator (Context) + Usage
<?php
// Context class — uses a strategy without knowing which one
// كلاس الـ Context — بيستخدم strategy من غير ما يعرف أيها
class ShippingCalculator
{
    private ShippingStrategyInterface $strategy;

    public function setStrategy(ShippingStrategyInterface $strategy): void
    {
        $this->strategy = $strategy;
    }
    public function calculate(float $weight, string $destination): array
    {
        return [
            'carrier' => $this->strategy->getCarrierName(),
            'cost'    => $this->strategy->calculateCost($weight, $destination),
            'days'    => $this->strategy->getEstimatedDays($destination),
        ];
    }
}

// Try all strategies / جرّب كل الـ strategies
$calc = new ShippingCalculator();
$options = [];
foreach ([new FedExStrategy(), new UPSStrategy(), new DHLStrategy()] as $s) {
    $calc->setStrategy($s);
    $options[] = $calc->calculate($weight, $destination);
}
usort($options, fn($a, $b) => $a['cost'] <=> $b['cost']);

When to Use Itمتى تستخدمه

✓ Good Use Cases✓ حالات جيدة

  • Multiple payment providers
  • Sorting/filtering algorithms at runtime
  • Export formats (PDF, CSV, Excel)
  • Pricing strategies (regular, wholesale, VIP)
  • Providers متعددة للـ payment
  • خوارزميات sorting/filtering وقت التشغيل
  • Formats تصدير (PDF, CSV, Excel)
  • استراتيجيات التسعير (عادي، جملة، VIP)

✗ Avoid When✗ تجنبه عندما

  • Only one algorithm that never changes
  • Simple if/else with 2 options is readable
  • خوارزمية واحدة فقط ما بتتغيرش
  • if/else بسيط بخيارين مقروء

When NOT to Use Itمتى لا تستخدمه

If you only have two options unlikely to grow, a simple if/else is more readable. Use Strategy when you have 3+ algorithms or when new ones need to be added frequently.لو عندك خيارين بس ومش هيكبروا، if/else بسيط أوضح. استخدم Strategy لما عندك 3+ خوارزميات أو لما الجديدة بتُضاف كتير.
